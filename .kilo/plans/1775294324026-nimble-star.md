# UltraTrader 專案架構

## 1. 專案用途

UltraTrader 是一個開源的台灣期貨（TAIFEX）演算法交易系統，使用 Python 開發。透過 Shioaji API 連接永豐證券執行自動化交易。

**核心目標：**
- 提供現成可用的台灣指數期貨（TMF、TGF 等）演算法策略
- 支援三種交易模式：**simulation**（本地模擬）、**paper**（永豐沙盒）、**live**（真實資金）
- 內建即時監控儀表板，可透過瀏覽器訪問
- 提供回測引擎進行歷史策略評估
- 實作完整的風險管理（斷路器、部位大小、回撤保護）

## 3. 核心模組

### `core/engine.py` - TradingEngine（系統核心）
- 事件驅動架構，背景執行緒運作主事件迴圈
- 管理多商品管道（每檔商品獨立的 TickAggregator、IndicatorEngine、Strategy）
- 從佇列處理事件：ticks 和完成的 K-bars
- 處理進出場執行，包含訂單冷卻和失敗重試機制
- 支援暖機：模擬模式產生合成 ticks，正式模式載入歷史 K-bars
- 整合所有子系統：Broker、MarketData、Strategy、Risk、Position、Performance、Intelligence

### `core/broker.py` - 券商抽象層
- `BaseBroker`：抽象介面（connect、disconnect、subscribe_tick、place_order 等）
- `ShioajiBroker`：真實實作，連接永豐證券 API，具備自動重連、心跳監控、token 過期偵測、成交確認
- `MockBroker`：本地模擬券商，產生具合理波動率的合成 tick 資料

### `core/market_data.py` - 市場資料處理
- `Tick`：即時 tick 資料結構
- `KBar`：OHLCV bar 結構
- `MarketSnapshot`：完整指標快照（EMA5/10/20/60/200、RSI、ATR、ADX、Bollinger Bands、K 棒形態、成交量分析）
- `TickAggregator`：將原始 ticks 轉換為多時間框架 K-bars（1m、5m、15m）
- `IndicatorEngine`：純 numpy/pandas 技術指標計算（無外部 TA 函式庫）

### `strategy/momentum.py` - Adaptive Momentum 策略（主要策略）
- 7 因子進場信號，多時間框架確認（5m/15m）
- 市場體制分類（TRENDING_UP、TRENDING_DOWN、RANGING、VOLATILE、CRISIS_DOWN、CRISIS_REVERSAL）
- 時段感知邏輯（開盤較高門檻、收盤前不新建倉）
- 7 層出場系統：收盤平倉、固定停損、保本停損、Chandelier Exit、獲利回吐保護、分層獲利、時間停損

### `risk/manager.py` - 4 層風險管理
- Layer 1（交易層）：Kelly 準則部位大小、停損合理性檢查
- Layer 2（日內層）：日虧損/交易次數限制
- Layer 3（帳戶層）：保證金檢查、最大回撤保護
- Layer 4（系統層）：斷路器、交易時段強制執行

### `dashboard/app.py` - Web 儀表板
- FastAPI 應用，提供 REST API 端點和 WebSocket 即時推送
- 端點：`/api/state`、`/api/trades`、`/api/kbars`、`/api/stats`、`/api/intelligence`、`/api/performance/*`、`/api/engine/{action}`、手動開平倉、模式切換
- 提供靜態 HTML 前端

### `backtest/engine.py` - 回測引擎
- 將歷史 K-bar 資料透過與正式交易相同的 Strategy + Risk 管道重放
- 包含滑價和手續費模擬
- 產生權益曲線、日 PnL、交易清單

## 4. 主要執行流程

**入口點：** `scripts/start.py`

```
1. 環境檢查（Python 3.10+、必要套件、.env 檔案）
2. 建立必要目錄（data/logs、data/historical、data/backtest_results）
3. 載入 .env 設定（TRADING_MODE、RISK_PROFILE、CONTRACT_CODE 等）
4. 初始化 TradingEngine：
   a. 從設定解析商品列表
   b. 為每檔商品建立 InstrumentPipeline（aggregator + indicator engine + strategy）
   c. 建立 Broker（simulation 用 MockBroker，paper/live 用 ShioajiBroker）
   d. 初始化 RiskManager、PositionManager、PerformanceTracker、Intelligence 模組
5. 建立 FastAPI 儀表板應用，綁定 engine
6. 啟動 uvicorn 伺服器於設定的 host:port（預設 127.0.0.1:8888）
7. 應用啟動時（FastAPI lifespan）：
   a. 啟動 WebSocket 佇列處理
   b. 連接券商
   c. 執行暖機（simulation 產生合成 ticks，live 載入歷史 K-bars）
   d. 訂閱即時 tick 資料
   e. 在背景執行緒啟動 engine 事件迴圈
   f. 啟動 intelligence 模組的資料收集器
8. 事件迴圈處理佇列中的事件：
   - Tick 事件 -> 更新 aggregator、檢查硬停損、廣播至儀表板
   - KBar 事件 -> 更新指標、執行策略邏輯（進出場）、執行交易
9. 關閉時（Ctrl+C）：強制平倉所有部位、儲存績效資料、斷開券商連線
```

**單商品資料流：**
```
Tick -> TickAggregator -> KBar complete -> IndicatorEngine -> MarketSnapshot
                                                    |
                                                    v
                                          Strategy.on_kbar() -> Signal
                                                    |
                                                    v
                                          RiskManager.evaluate() -> RiskDecision
                                                    |
                                                    v
                                          Broker.place_order() -> OrderResult
                                                    |
                                                    v
                                          PositionManager.open/close_position()
```

## 5. 技術棧

| 類別 | 技術 |
|---|---|
| **語言** | Python 3.10+ |
| **券商 API** | Shioaji >= 1.2.0（永豐證券） |
| **Web 框架** | FastAPI >= 0.115.0 |
| **Web 伺服器** | Uvicorn >= 0.34.0 |
| **即時通訊** | WebSockets >= 13.0 |
| **資料處理** | pandas >= 2.2.0, numpy >= 1.26.0 |
| **日誌** | loguru >= 0.7.0 |
| **環境設定** | python-dotenv >= 1.0.0 |
| **HTTP 客戶端** | requests >= 2.31.0 |
| **市場資料** | yfinance >= 0.2.36 |
| **架構** | 事件驅動、多執行緒（背景 engine 執行緒 + WebSocket 任務） |
| **測試** | Python unittest（tests/ 目錄） |
| **授權** | MIT |

**重要設計決策：**
- 零外部技術分析函式庫 — 所有指標（EMA、RSI、ATR、ADX、Bollinger Bands）均使用純 numpy/pandas 從頭實作
- 多商品支援，每檔商品獨立管道
- 三階段交易 progression：simulation -> paper -> live
- 風險狀態持久化至磁碟（重開機後存活）
- 正式交易具備自動重連和心跳監控
- 訂單冷卻機制防止 API 速率限制濫用
