# 自動發文功能待開發

本文件紀錄了從 `PERFORMANCE_SPEC.md` 中獨立出來的自動發文功能需求，將於績效追蹤系統開發且穩定後，再行實作。

## 待開發功能摘要
建置 API 接口，讓系統在交易收盤或結算時，能夠自動抓取交易績效數據並生成每日 Threads 文案，最後將資料推播至 MindThread Firebase。

## 預計實作項目

### 1. 新增 MindThread 串接服務 (`dashboard/services/mindthread.py`)
- 實作與 MindThread (Firebase Firestore) 通訊的服務層。
- 負責將每日結算文案與績效資料上傳至 `library/{docId}` 集合。

### 2. 擴充 Dashboard API (`dashboard/app.py`)
- 註冊端點 `/api/publish/push-to-mindthread`。
- API 被呼叫時應：
  1. 呼叫底層績效模組生成當日總結文案。
  2. 透過 `mindthread.py` 寫入至 Firebase。
  3. 回傳成功狀態與生成內容。

### 3. 環境與依賴配置 (`requirements.txt` / `.env`)
- 依據開發決策，將 `firebase-admin` (或其他 API 串接套件) 加入至專案相依套件中。
- 在 `.env` 需讀取以下環境變數作為啟用判斷與驗證：
  - `MINDTHREAD_ENABLED=true`
  - `MINDTHREAD_ACCOUNT_ID`
  - `MINDTHREAD_USER_ID`
  - `FIREBASE_CREDENTIALS` (Service Account JSON 路徑)

### 4. 系統串接整合 (`core/engine.py` / `core/performance.py`)
- 當 `TradingEngine` 觸發 `on_session_end` 且完成績效結算後。
- 若 `MINDTHREAD_ENABLED` 為 `true`，自動觸發 Push 行為至 MindThread 平台。
- 需撰寫防呆機制，確認每日僅發送一次，避免重複推文。
